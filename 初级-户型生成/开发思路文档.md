# 户型生成 Revit 2026 插件 — 开发思路文档

## 一、需求概述

在建筑方案推演阶段，设计师需要快速由矩形构成的可网页端设计的户型功能。网页设计数据（JSON格式）需在 Revit 中自动创建**墙体、房间、房间标记**三种元素，并按照现实情况进行处理：
- 内外墙厚度不同（内墙 100mm，外墙 200mm）
- 同方向同厚度墙体贯通（合并为一条连续墙）
- 外墙边界朝户型外侧扩大 100mm

## 二、架构设计

### 2.1 整体架构

采用分层架构，职责清晰：

```
┌─────────────────────────────────────────────┐
│           App.cs (IExternalApplication)      │ ← Ribbon 面板注册
├─────────────────────────────────────────────┤
│           Command.cs (IExternalCommand)      │ ← 主命令入口
├─────────────────────────────────────────────┤
│           FloorPlanService.cs                │ ← 编排服务
├─────────────────┬───────────────────┬───────┤
│  WallService.cs  │  RoomService.cs    │ JsonReader.cs │
│  (墙体算法核心)  │  (房间+标记生成)   │ (JSON解析)    │
├─────────────────┴───────────────────┴───────┤
│          Models (RoomData / RoomJson)         │ ← 数据模型
└─────────────────────────────────────────────┘
```

### 2.2 数据模型

**RoomData** 类表示一个房间：
- `CentroidX / CentroidY` — 房间中心点
- `Name` — 房间名称
- `Bay / Depth` — 开间（水平长度）/ 进深（垂直长度）
- `GetBoundaryLines()` — 计算四个边界线（左/右/下/上）

**BoundaryLine** 类表示一条墙中心线：
- `Start / End` — 起终点坐标
- `Orientation` — 方向（水平/垂直）
- `IsLeft` — 对于垂直墙标识左/右，水平墙标识上/下
- `IsExterior` — 是否为外墙
- `ThicknessMm` — 对应墙厚度

## 三、核心算法

### 3.1 内外墙判定

**思路**：遍历所有房间的所有边界线，对每一对来自不同房间、方向相同的边界线，判断是否**共线且有重叠**。如果存在这样的配对，说明两房间共享该墙，标记为**内墙**（100mm）；否则该墙为某个房间独有，标记为**外墙**（200mm）。

**判断共线重叠条件**（以垂直墙为例）：
1. X 坐标差 < 容差 → 共线
2. Y 方向区间有交集 → 重叠

### 3.2 外墙外扩

外墙边界线向外侧偏移 100mm：
- 左墙 → X 减小
- 右墙 → X 增大
- 下墙 → Y 减小
- 上墙 → Y 增大

### 3.3 墙体合并贯通

**目的**：将同方向、同厚度的共线墙体合并为一条连续墙。

**算法步骤**：
1. 按 `(方向, 厚度, 固定坐标)` 分组
2. 每组内收集一维区间（垂直墙为 Y 区间，水平墙为 X 区间）
3. 区间排序后合并重叠/相接的区间
4. 每个合并后的区间创建一条连续的 Revit 墙体

### 3.4 房间与标记生成

在每房间中心点调用 `Document.Create.NewRoom()` 创建 Room，设置名称后在同一位置创建 `RoomTag`。

## 四、用户交互流程

```
Command.Execute()
  │
  ├─ 检查当前视图是否为 ViewPlan（否则提示用户）
  ├─ 弹出 OpenFileDialog 选择 huxi.json
  ├─ JsonReader 解析 JSON
  ├─ FloorPlanService.GenerateFloorPlan()
  │   ├─ 清空已有户型（墙体 + 房间 + 标记）
  │   ├─ WallService.CreateWalls()  → 创建墙体
  │   └─ RoomService.CreateRoomsAndTags()  → 创建房间 + 标记
  └─ TaskDialog 显示完成信息
```

## 五、关键技术点

| 技术点 | 说明 |
|--------|------|
| Revit API 版本 | Revit 2026，引用 RevitAPI.dll + RevitAPIUI.dll |
| 事务管理 | 每个操作独立事务（清空/建墙/建房间），实现原子性 |
| 单位转换 | 毫米 → 英尺（除以 304.8），定义 MmToFeet() 扩展方法 |
| 墙体类型 | 通过 Duplicate 系统默认墙类型创建内墙/外墙专用类型 |
| 坐标系统 | 使用房间中心点 ± 开间/进深 计算四边，Z=0（当前标高） |

## 七、部署与使用

1. 在 Visual Studio 2022 中以 .NET Framework 4.8 编译项目
2. 将 `FloorPlanGenerator.addin` + `FloorPlanGenerator.dll` 放入：
   `%APPDATA%\Autodesk\Revit\Addins\2026\`
3. 启动 Revit 2026，打开楼层平面视图
4. 点击 "户型生成" 标签 → "生成户型" 按钮
5. 选择 `huxi.json` 文件，自动生成户型
