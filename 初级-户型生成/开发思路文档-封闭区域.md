# 封闭区域检测 Revit 2026 插件 — 开发思路文档

## 一、需求概述

在 Revit 楼层平面视图中，用户绘制白色详图线构成复杂图形（含直线、圆弧、样条曲线）。需通过算法自动检测所有封闭区域，生成两种结果：
- **最大封闭区域**：合并相邻最小区域，取面积最大的 3 个，黄色填充
- **最小封闭区域**：所有独立的封闭区域，洋红色填充

要求不使用 Revit 原生房间功能，使用自定义数据结构实现平面图环检测算法。

## 三、核心算法：平面图环检测

### 3.1 算法流程

```
输入详图线集合
    ↓
① 曲线预处理（去重、求交、分割）
    ↓
② 构建图结构（GraphVertex + GraphEdge）
    ↓
③ 半边遍历查找所有面（最小封闭区域）
    ↓
④ 合并相邻面得到最大封闭区域
    ↓
⑤ 生成 FilledRegion
```

### 3.2 曲线预处理（CurveProcessor）

- **去重**：检测端点匹配的重复曲线，移除重叠
- **求交**：使用 Revit `Curve.Intersect()` API 检测所有交点
- **分割**：在交点处分割曲线，保留原始曲线类型（直线、弧线、样条）

### 3.3 图构建

每条分割后的曲线段成为一条 `GraphEdge`，端点为 `GraphVertex`。构建邻接表。

### 3.4 半边遍历找面（FaceDetector）

采用**半边数据结构 + 最小角遍历**算法：

```
对每条边(GraphEdge)的两个方向（正向/反向）：
  ① 从该半边出发
  ② 在当前顶点找到出射角最小的下一条边
  ③ 继续遍历直到回到起点
  ④ 若步数边数 ≥ 3 则形成一个封闭面
```

- 顺时针遍历 = 外环（无限大区域），丢弃
- 逆时针遍历 = 内环（有效封闭区域），保留

### 3.5 面合并

相邻面（共享边的面）通过 BFS 分组合并为最大区域。合并后移除内部共享边，仅保留边界边。

### 3.6 数值鲁棒性

- 容差设为 0.0164 英尺（约 5mm）
- 交点判定、端点匹配均使用该容差
- 极小面（面积 < 容差²）自动过滤

## 四、用户交互流程

### MaxRegionCommand
```
用户选中所有白色详图线 → 曲线预处理 → 环检测 → 面合并
→ 删除上次填充区域 → 生成黄色 FilledRegion（最大3个）
```

### MinRegionCommand
```
用户选中所有白色详图线 → 曲线预处理 → 环检测
→ 删除上次填充区域 → 生成洋红色 FilledRegion（所有最小面）
```

## 五、关键技术点

| 技术点 | 说明 |
|--------|------|
| Revit 版本 | Revit 2026，引用 RevitAPI.dll + RevitAPIUI.dll |
| 事务管理 | 每个命令独立事务 |
| 曲线类型 | 直线、圆弧、样条曲线均支持 |
| 特殊场景 | T形/十字连接、孤岛区域、重叠曲线 |
| 重复执行 | 自动删除上次生成的填充区域 |

## 七、部署与使用

1. 在 VS 2022 中以 .NET Framework 4.7.2 编译
2. 将 `ClosedRegion.dll` + `ClosedRegion.addin` 放入 `%APPDATA%\Autodesk\Revit\Addins\2026\`
3. 打开 `封闭区域-测试用例.rvt`，进入"标高1"视图
4. 输入 `MaxRegionCommand` → 框选白色曲线 → 生成最大封闭区域
5. 输入 `MinRegionCommand` → 框选白色曲线 → 生成最小封闭区域
