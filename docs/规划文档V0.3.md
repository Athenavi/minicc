# MiniCC V0.3 — AI Agent 编排平台

> **愿景**：打造超越 LangGraph + Dify 的 AI Agent 编排平台。LangGraph 提供图状工作流引擎，Dify 提供可视化 RAG 平台。MiniCC V0.3 将两者合二为一，加上 V0.1 的代码智能 + V0.2 的 RPA 自动化，成为统一的 AI 生产力平台。

> **一句话定位**：V0.1 是"能写代码的 AI"，V0.2 是"能操作数字系统的 AI"，V0.3 是"能编排一切 AI 能力的平台"。

## 2. V0.3 核心架构

```
用户
  │
  ├── 可视化编排器 (Dify 式画布)
  │   ├── 拖拽节点 (LLM / Tool / Agent / RAG / Code / Condition)
  │   ├── 边线连接 (数据流 / 控制流)
  │   └── 实时预览 (运行中节点高亮)
  │
  ├── StateGraph 引擎 (LangGraph 式)
  │   ├── 编译时 Schema → Pydantic
  │   ├── 节点执行 (支持并行)
  │   ├── 条件边 / 循环边
  │   ├── 状态检查点 (Checkpoint)
  │   └── 流式输出 (每个节点)
  │
  ├── RAG 管道
  │   ├── 多源文档加载器 (PDF/Web/DB/Code)
  │   ├── 分块策略 (Semantic / Token / Recursive)
  │   ├── 向量存储 (Chroma / PGVector / Redis)
  │   ├── 检索策略 (Hybrid / Rerank / Multi-Query)
  │   └── Agent Rerank (LLM 排序结果)
  │
  ├── Agent 集群
  │   ├── 路由 Agent (任务分发)
  │   ├── 代码 Agent (V0.1)
  │   ├── RPA Agent (V0.2)
  │   ├── 知识 Agent (RAG)
  │   └── 工具 Agent (MCP/API)
  │
  ├── 监控与可观测性
  │   ├── 全链路追踪 (OpenTelemetry)
  │   ├── Token 消耗统计
  │   ├── 节点级耗时
  │   ├── 成功率/延迟仪表盘
  │   └── 调试回放
  │
  └── 平台化
      ├── REST API 暴露 (创建/执行/监控 Agent)
      ├── 插件市场
      ├── 多租户
      └── Webhook 触发器
```

## 3. 新增模块与技术选型

| 模块 | 技术选型 | 对标 |
|:-----|:---------|:-----|
| **StateGraph 引擎** | 自研 `Pydantic` + `asyncio` | LangGraph StateGraph |
| **可视化编排器** | `ReactFlow` + `React` | Dify 画布 |
| **RAG 管道** | `Chroma` / `pgvector` + `LangChain` 分块 | Dify 知识库 |
| **向量存储** | `Chroma`（开发）/ `PGVector`（生产）| Dify 向量化 |
| **文档解析** | `Unstructured` / `PyMuPDF` / `MarkItDown` | Dify 文档加载 |
| **重排序** | `Cohere Rerank` / `LLM Rerank` | Dify Rerank |
| **追踪系统** | `OpenTelemetry` + `Logfire` / `Langfuse` | LangSmith |
| **多模态** | `Playwright` 截图 + LLM Vision | GPT-4V |
| **检查点** | SQLite + Redis 持久化 | LangGraph Checkpoint |

## 5. V0.3 开发阶段（6 Phase，~16 周）

### Phase G: StateGraph 引擎（4 周）
**目标**：实现 LangGraph 级别的图状工作流引擎。

| 任务 | 产出 | 说明 |
|:-----|:-----|:-----|
| **G.1** Graph 定义 | `graph/graph.py` | Nodes + Edges + StateGraph Schema |
| **G.2** 状态管理 | `graph/state.py` | Pydantic 编译时 Schema，Reducder 支持 |
| **G.3** 图执行器 | `graph/executor.py` | 拓扑排序、并行节点、条件边 |
| **G.4** 检查点 | `graph/checkpoint.py` | 每个节点执行后保存状态，支持恢复 |
| **G.5** 流式输出 | `graph/streaming.py` | 节点级流式推送（SSE） |
| **G.6** 工具/Agent 节点适配 | 集成现有工具 | 将 V0.2 工作流升级为 StateGraph |

### Phase H: RAG 管道（3 周）
**目标**：实现 Dify 级别的知识库检索增强生成。

| 任务 | 产出 | 说明 |
|:-----|:-----|:-----|
| **H.1** 文档加载器 | `rag/loader.py` | PDF/HTML/Office/Web 加载 |
| **H.2** 分块策略 | `rag/chunker.py` | Recursive/Semantic/Token 分块 |
| **H.3** 向量化 | `rag/embedder.py` | 多 Provider 嵌入（OpenAI/HuggingFace） |
| **H.4** 检索策略 | `rag/retriever.py` | Hybrid/Sparse/Dense/Multi-Query |
| **H.5** 重排序 | `rag/reranker.py` | Cohere/LLM/Cross-Encoder |
| **H.6** 向量存储 | `rag/vector_store.py` | Chroma/PGVector/Redis 适配 |

### Phase I: Agent 集群（2 周）
**目标**：多 Agent 协同，路由分发。

| 任务 | 产出 |
|:-----|:-----|
| **I.1** 路由 Agent | `agents/router.py` — 任务分发 |
| **I.2** 知识 Agent | `agents/knowledge_agent.py` — RAG 查询 |
| **I.3** Agent 通信总线 | Agent 间消息传递 |

### Phase J: 可视化编排器（3 周）
**目标**：Dify 风格的拖拽式工作流设计器。

| 任务 | 产出 |
|:-----|:-----|
| **J.1** ReactFlow 画布 | `Canvas.tsx` — 拖拽/连接/缩放 |
| **J.2** 节点面板 | `NodePanel.tsx` — LLM/Tool/Agent/RAG/Code |
| **J.3** 配置面板 | `Settings.tsx` — 节点参数编辑 |
| **J.4** 实时预览 | 运行中节点高亮 + 流式输出 |

### Phase K: 可观测性（2 周）
**目标**：LangSmith 级别的全链路追踪。

| 任务 | 产出 |
|:-----|:-----|
| **K.1** OpenTelemetry 集成 | `observability/tracer.py` |
| **K.2** 追踪存储 | SQLite/Redis 持久化 |
| **K.3** 监控仪表盘 | 前端 TraceView |

### Phase L: 平台化（2 周）
**目标**：API 暴露 + 多租户 + 插件市场。

| 任务 | 产出 |
|:-----|:-----|
| **L.1** REST API | Agent/Graph/Monitor CRUD |
| **L.2** Webhook | 事件触发 |
| **L.3** 多租户 | 团队隔离 |
| **L.4** 插件市场 | 社区工具分享 |

## 7. 启动顺序

```
Phase G (StateGraph) → Phase H (RAG) → Phase I (Agent Cluster)
    → Phase J (Visual Editor) → Phase K (Observability) → Phase L (Platform)
```
