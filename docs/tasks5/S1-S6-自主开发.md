# S1-S6: 自主开发流水线

> **总预估**：6.5 周 | **前置**：Phase R

## S1: 架构 Agent 增强（1 周）

`backend/app/agents/architect.py`：
- 读取 PRD + 技术方案
- 生成模块结构（目录树）
- 定义接口（函数签名、类结构）
- 输出数据流图
- 与 V0.3 Graph 集成：架构输出 → Graph 节点定义

## S2: 编码 Agent 集群（2 周）

`backend/app/agents/coding_cluster.py`：

```python
class CodingCluster:
    """多 Agent 并行编码集群。"""

    async def execute(self, tasks: list[GraphTask]) -> list[TaskResult]:
        """为每个模块创建一个子 Agent：
        - Agent A: 模块 A（数据层）
        - Agent B: 模块 B（业务层）
        - Agent C: 模块 C（展示层）
        - 每个 Agent 使用 V0.4 Editor 工具
        """
```

### 关键设计

- 每个子 Agent 使用独立 EditorSession
- 共享接口定义（通过 Memory）
- 冲突检测（复用 V0.4 O4）
- 进度追踪（任务状态 → 前端展示）

## S3: 测试 Agent（1 周）

`backend/app/agents/tester.py`：

```python
class TestAgent:
    """自动生成并运行测试。"""

    async def test_module(self, module_path: str) -> TestReport:
        """1. 分析模块代码 → 生成测试用例
           2. 生成 pytest unittest 代码
           3. 运行测试 → 收集结果
           4. 失败 → AI 分析根因
        """
```

### 测试类型

- 单元测试（pytest）
- 集成测试（模块间）
- E2E 测试（Playwright，复用 V0.2）
- 覆盖率报告

## S4: Review Agent（1 周）

`backend/app/agents/reviewer.py`：

```python
class ReviewAgent:
    """代码审查 + 安全审计 + 性能分析。"""

    async def review(self, pr_changes: list[FileDiff]) -> ReviewReport:
        """检查：
        - 代码风格（PEP 8 / ESLint）
        - 安全漏洞（SQL 注入、XSS）
        - 性能问题（N+1 查询、内存泄漏）
        - 测试覆盖
        - 文档完整性
        """
```

## S5: 自动修复（1 周）

`backend/app/agents/autofix.py`：
- Review 发现问题 → 生成修复方案
- 应用修复（通过 V0.4 Editor）
- 重新 Review → 循环直到通过
- 最大循环次数：3 次

## S6: 合并管理（0.5 周）

`backend/app/agents/merger.py`：
- 自动解决冲突（复用 V0.4 O4）
- 生成 Merge Commit
- 创建 PR（复用 V0.4 O3）
- CI 通过后自动 Merge

### 验收标准
- [ ] 架构 Agent 生成模块结构
- [ ] 多 Agent 并行编码无冲突
- [ ] 测试覆盖 > 80%
- [ ] Review 自动发现问题
- [ ] 自动修复循环
- [ ] 自动创建并合并 PR
- [ ] 120 测试通过
