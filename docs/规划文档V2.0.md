# MiniCC V2.0 总体规划文档 — Go 重构

> **核心理念**：用 Go 从零重构 MiniCC，原生解决 10 个阻塞级约束，支撑百万 DAU。
> **工程哲学**：ponytail — YAGNI、stdlib 优先、删除先于添加、一行够用不写两行。

---

## 1. 为什么重构

### 1.1 Python 版现状

Python 版 MiniCC 已验证了产品形态（233+ 工具、10 版本演进、52 模块），但架构上无法支撑生产规模：

| 约束 | Python 现状 | 根因 |
|:-----|:-----------|:-----|
| 13 个独立 SQLite | 每模块自建 DB，同步阻塞 | 无统一数据层 |
| 内存 Dict 状态 | TaskManager / Engine / Permission 全在进程内存 | 无状态管理设计 |
| 进程内 Broadcaster | asyncio.Queue 无法跨实例 | SSE 设计未考虑分布式 |
| 无认证 | CORS 全开，任何请求放行 | 原型期未引入 |
| 无限流 | 单用户可以打满全部资源 | 未设计防护 |
| 单 LLM Provider | 一个 API Key 一个模型 | 硬编码配置 |
| 任务无追踪 | create_task 后无人管理 | 异步无基础设施 |
| 本地文件系统 | 所有路径解析到本地磁盘 | 未抽象存储层 |
| 无分布式缓存 | 每个进程独立 LRU Dict | 缓存设计缺失 |
| 单进程部署 | uvicorn 单 worker | 从未考虑水平扩展 |

### 1.2 为什么是 Go

| 维度 | Python | Go | ponytail 匹配度 |
|:-----|:-------|:---|:----------------|
| 并发模型 | asyncio（协作式，GIL 约束） | goroutine（抢占式，P 模型） | ✅ 原生并发免框架 |
| 部署 | 需要 Python 解释器 + pip + 虚拟环境 | 静态编译单二进制 | ✅ 零依赖部署 |
| 启动时间 | 300-800ms（含导入） | <10ms | ✅ 快速伸缩 |
| 内存 | 100-300MB（基础进程） | 10-30MB（基础进程） | ✅ 低成本 |
| 标准库 | HTTP/JSON/DB 需三方库 | net/http + encoding/json + database/sql 即用 | ✅ stdlib 优先 |
| 错误处理 | 异常（隐式传播） | error（显式处理） | ✅ 不隐藏问题 |
| 工具生态 | pip install（版本冲突） | go build（静态链接） | ✅ 无依赖地狱 |
| 交叉编译 | 复杂 | `GOOS=linux GOARCH=arm64 go build` | ✅ 简单 |

## 4. 10 个阻塞级约束的 Go 原生解决方案

### C1: 13 个独立 SQLite → 统一 PostgreSQL

```go
// internal/db/postgres.go — 单一连接池，全局共享
package db

import (
    "context"
    "github.com/jackc/pgx/v5/pgxpool"
)

var Pool *pgxpool.Pool

func Connect(ctx context.Context, dsn string) error {
    cfg, _ := pgxpool.ParseConfig(dsn)
    cfg.MaxConns = 50              // 连接池上限
    cfg.MinConns = 10              // 最小空闲
    cfg.MaxConnLifetime = 30 * 60  // 30分钟回收
    cfg.HealthCheckPeriod = 60     // 每秒健康检查
    var err error
    Pool, err = pgxpool.NewWithConfig(ctx, cfg)
    return err
}

// 所有模块使用同一个 Pool，不再各自建 DB
// 每个 handler 从池中获取连接，用完归还
```

**ponytail 决策**：不用 ORM（GORM/Ent）。`pgx` 是 Go 生态最接近 stdlib 的 pg 驱动，直接写 SQL。查询用 `sqlc` 生成类型安全代码，避免手写字符串拼接。

### C2: 内存 Dict 状态 → PostgreSQL 任务表 + Redis Session

```go
// internal/engine/query.go — 无状态 Engine
type QueryEngine struct {
    SessionID string
    UserID    string
    // 不持有任何运行时状态
    // 每次请求从 Redis/PostgreSQL 重建上下文
}

// internal/session/manager.go — Session 管理
type SessionManager struct {
    redis *redis.Client
    pg    *pgxpool.Pool
}

func (m *SessionManager) GetSession(ctx context.Context, id string) (*model.Session, error) {
    // 1. Redis 热数据 (TTL 2h)
    data, err := m.redis.Get(ctx, "session:"+id).Bytes()
    if err == nil {
        var s model.Session
        return &s, json.Unmarshal(data, &s)
    }
    // 2. PostgreSQL 冷数据
    return m.loadFromPG(ctx, id)
}
```

### C3: 进程内 Broadcaster → Redis Pub/Sub

```go
// internal/broadcast/redis_pubsub.go
type RedisHub struct {
    rdb *redis.Client
    pubsub *redis.PubSub
}

func (h *RedisHub) Publish(channel string, event Event) error {
    data, _ := json.Marshal(event)
    return h.rdb.Publish(ctx, channel, data).Err()
}

func (h *RedisHub) Subscribe(channel string) <-chan Event {
    ch := make(chan Event, 64)
    go func() {
        sub := h.rdb.Subscribe(ctx, channel)
        for msg := range sub.Channel() {
            var e Event
            json.Unmarshal([]byte(msg.Payload), &e)
            select {
            case ch <- e:
            default: // 慢客户端丢弃
            }
        }
    }()
    return ch
}
```

### C4: 无认证 → JWT + API Key + RBAC

```go
// internal/auth/jwt.go
type JWTClaims struct {
    UserID   string   `json:"uid"`
    Role     string   `json:"role"`  // owner / admin / user
    Perms    []string `json:"perms"` // ["chat:write", "admin:read"]
    jwt.RegisteredClaims
}

// internal/api/middleware.go
func AuthMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        // 优先级: API Key Header > Bearer Token > Cookie
        token := extractToken(r)
        claims, err := jwt.ParseWithClaims(token, &JWTClaims{}, keyFunc)
        if err != nil {
            http.Error(w, "unauthorized", http.StatusUnauthorized)
            return
        }
        ctx := context.WithValue(r.Context(), "user", claims)
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}
```

### C5: 无限流 → Token Bucket 多维度限流

```go
// internal/llm/ratelimit.go
type RateLimiter struct {
    redis *redis.Client
}

// 三层限流: 用户级 + 模型级 + 全局
func (rl *RateLimiter) Allow(ctx context.Context, userID, model string) bool {
    keys := []string{
        "ratelimit:user:" + userID,     // 用户: 100 rpm
        "ratelimit:model:" + model,     // 模型: 1000 rpm
        "ratelimit:global",              // 全局: 10000 rpm
    }
    for _, key := range keys {
        val, _ := rl.redis.Incr(ctx, key).Result()
        if val == 1 {
            rl.redis.Expire(ctx, key, time.Minute)
        }
        if val > getLimit(key) {
            return false
        }
    }
    return true
}
```

### C6: 单 LLM Provider → 多 Provider 网关

```go
// internal/llm/gateway.go
type Gateway struct {
    providers []Provider           // 按优先级排序
    cache     *Cache
    breaker   *CircuitBreaker
    fallback  *FallbackStrategy
}

func (g *Gateway) Chat(ctx context.Context, req *Request) (*Response, error) {
    // 1. 语义缓存命中？直接返回
    if cached := g.cache.Get(req); cached != nil {
        return cached, nil
    }
    // 2. 按优先级尝试每个 provider
    for i, p := range g.providers {
        if !g.breaker.Allow(p.Name()) {
            continue // 熔断中，跳过
        }
        resp, err := p.Chat(ctx, req)
        if err == nil {
            g.cache.Set(req, resp)
            return resp, nil
        }
        g.breaker.Fail(p.Name())
        // 降级到下一个
    }
    // 3. 全失败 → 返回降级响应
    return g.fallback.Response(ctx, req)
}
```

### C7: 任务无追踪 → Redis Streams + 任务表持久化

```go
// internal/queue/redis_streams.go
type TaskQueue struct {
    rdb *redis.Client
}

func (q *TaskQueue) Enqueue(ctx context.Context, task *Task) (string, error) {
    data, _ := json.Marshal(task)
    id, err := q.rdb.XAdd(ctx, &redis.XAddArgs{
        Stream: "tasks:" + task.Type,
        Values: []string{"data", string(data)},
        MaxLen: 100000,
    }).Result()
    if err != nil {
        return "", err
    }
    // 写 PostgreSQL 任务表做持久化备份
    q.persistTask(ctx, task)
    return id, nil
}

// Worker 从消费者组消费
func (w *Worker) Start(ctx context.Context) {
    for {
        msgs, err := w.rdb.XReadGroup(ctx, &redis.XReadGroupArgs{
            Group:    "workers",
            Consumer: w.id,
            Streams:  []string{"tasks:llm", ">"},
            Count:    1,
            Block:    5 * time.Second,
        }).Result()
        // ... 处理消息
        // 完成后 XACK
        w.rdb.XAck(ctx, "tasks:llm", "workers", msg.ID)
    }
}
```

### C9: 无分布式缓存 → Redis 缓存层

```go
// internal/cache/redis_cache.go
type Cache struct {
    rdb    *redis.Client
    ttl    time.Duration
    prefix string
}

func (c *Cache) Get(ctx context.Context, key string) ([]byte, bool) {
    data, err := c.rdb.Get(ctx, c.prefix+key).Bytes()
    return data, err == nil
}

func (c *Cache) Set(ctx context.Context, key string, value []byte) {
    c.rdb.Set(ctx, c.prefix+key, value, c.ttl)
}
```

| GORM / Ent | ORM | ❌ 直接写 SQL，pgx 够用 |
| Wire / Uber-FX | DI 容器 | ❌ 手动依赖注入，清晰可见 |
| Viper | 配置 | ❌ 环境变量 + 小 config.go 即可 |

**最终外部依赖 ≤ 10 个**。

## 7. 估算

| Phase | 周 | 产出 | DAU 支撑 |
