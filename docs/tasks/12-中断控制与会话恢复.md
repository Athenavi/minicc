# 任务 12：中断控制与会话恢复

> **所属阶段**：Phase 3 - 状态持久化与任务系统 (State & Task)
> **对应模块**：模块 7 (Interrupt/Cancel + Session Reconnect)
> **预估工时**：2-3 天
> **依赖**：任务 03 (WebSocket 协议)、任务 11 (Session/TaskManager)

## 1. 任务目标

实现完整的任务中断控制机制：前端"停止生成"按钮 → WebSocket CancelSignal → 后端协程取消 → 资源清理。同时确保会话重连时状态完全恢复，用户体验无缝。

## 2. 详细子任务

### 2.1 中断信号链

```
前端点击 "停止生成"
  │
  ├── WebSocket.send({ type: "cancel" })
  │
  ▼
后端 WebSocket reader 协程收到消息
  │
  ├── QueryEngine.abort_event.set()   ← 协作式中断信号
  │
  ├── TaskGroup 中的运行中工具收到取消
  │   └── 正在执行的 FileSystem/Shell 操作收到 CancelledError
  │
  ├── 所有进行中的 PermissionRequest 被标记为拒绝
  │
  └── 当前消息组装完毕，发送 message_complete(interrupted=true)
      └── 前端显示 "⏸️ 已中断" 标记
```

### 2.2 后端协程中断实现

- [ ] 文件：`backend/app/engine/query_engine.py`（扩展）

```python
class QueryEngine:
    def __init__(self, ...):
        self.abort_event = asyncio.Event()
        self._current_tool_task: asyncio.Task | None = None
    
    async def cancel(self):
        """
        取消当前正在执行的操作。
        协作式取消——设置信号，等待协程自行退出。
        """
        self.abort_event.set()
        
        # 取消正在执行的工具
        if self._current_tool_task and not self._current_tool_task.done():
            self._current_tool_task.cancel()
        
        # 通知所有等待审批的请求
        if self.permission_handler:
            await self.permission_handler.cancel_all_pending()
    
    async def _run_tool_with_cancel(self, tool_call, tool) -> ToolResult:
        """执行工具调用，支持取消"""
        self._current_tool_task = asyncio.current_task()
        try:
            return await tool.execute(validated_input)
        except asyncio.CancelledError:
            # 工具被取消——清理资源
            return ToolResult(
                tool_call_id=tool_call.id,
                output="操作已被用户中断",
                is_error=True,
            )
        finally:
            self._current_tool_task = None
```

### 2.5 会话重连完整流程

```
用户断开 → 浏览器关闭 / 网络断开
  │
  ├── 后端检测到 WebSocket 断开
  │   ├── TaskManager 继续保持会话任务（最多 30 分钟）
  │   ├── PermissionHandler 挂起所有待审批请求
  │   └── 后台执行的任务继续
  │
用户重连 → 重新打开页面 / 网络恢复
  │
  ├── 前端 useWebSocket 自动重连
  │   ├── 指数退避重试
  │   └── 携带 session_id
  │
  ├── 后端收到 session_resume
  │   ├── TaskManager 查找已有会话
  │   │   ├── 找到 → 复用 QueryEngine
  │   │   └── 未找到 → SessionManager.resume_session()
  │   │
  │   ├── 重放最近 N 条消息（含工具调用卡片状态）
  │   ├── 恢复审批等待状态（如果有的话）
  │   └── 发送当前 Agent 状态（thinking/executing/waiting_approval）
  │
  └── 前端恢复所有卡片状态
```

## 5. 注意事项

- 中断必须是**协作式**的——不能硬杀线程，避免资源泄露
- `TaskGroup` 是 Python 3.14 的结构化并发推荐方式
- 重连时消息顺序必须严格保持一致，不能出现乱序
- 后台任务在用户重连前可能已经完成——重连时应直接发送最终结果
- 停止按钮在工具执行期间必须可见，且响应迅速
