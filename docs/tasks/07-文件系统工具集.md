## 1. 任务目标

实现 AI 读写和编辑文件的核心工具集，包括 `read_file`（带行号）、`write_to_file`（创建/覆盖）、`str_replace_editor`（精确替换编辑）等操作，配备路径安全沙箱和 Diff 预览能力。

            Path("/etc"), Path("/sys"), Path("/proc"),
            Path(".git"),  # 禁止直接修改 .git 目录

        - 检查是否在 blocked_paths 中
        - 返回解析后的 Path 对象（合法时）
        - 抛出 PermissionError（不合法时）

### 2.4 `StrReplaceEditorTool` — 精确替换编辑

- [ ] 文件：`backend/app/tools/file_system.py`

```python
class StrReplaceEditorInput(BaseModel):
    path: str = Field(description="文件路径")
    old_string: str = Field(description="需要替换的精确字符串（必须唯一匹配）")
    new_string: str = Field(description="替换后的字符串")
    replace_all: bool = Field(default=False, description="是否替换所有匹配项")

class StrReplaceEditorTool(BaseTool):
    name = "str_replace_editor"
    description = "在文件中进行精确的字符串替换编辑。适用于修改代码中的特定行或函数。"
    input_schema = StrReplaceEditorInput
    permission_level = PermissionLevel.WRITE  # 需要审批
```

- [ ] 核心逻辑：
  - 读取文件 → 用 `old_string` 查找匹配 → 确认唯一性 → 替换 → 写回
  - 如果 `old_string` 匹配 0 次 → 返回错误
  - 如果 `old_string` 匹配 >1 次且 `replace_all=False` → 返回错误（要求更精确的匹配或启用 replace_all）
- [ ] `diff_preview`：使用 `difflib.unified_diff` 生成统一 diff 格式

## 5. 注意事项

- **路径安全是第一优先级**——必须防止任意路径读写漏洞
- 写入操作使用原子写入（write to temp → rename），防止断电/崩溃导致文件损坏
- `StrReplaceEditorTool` 的 `old_string` 必须要求前后文唯一性，这是避免编辑错行的关键
- 文件读取超过 1MB 时截断并提示，防止炸屏
- 行号格式必须对齐开发者的阅读习惯：`    42│code`
