## 1. 任务目标

实现完整的权限与审批状态机：工具调用被 LLM 发起时，主循环挂起，通过 WebSocket 向前端发送审批请求，等待用户批准/拒绝/始终允许，然后继续执行或中止。

这是 MiniCC 安全体系的核心——"先审批后执行"的看门狗。

## 2. 详细子任务

### 2.1 权限等级定义

- [ ] 文件：`backend/app/core/permission.py`

```python
class PermissionLevel(IntEnum):
    """权限等级（数值越高越严格）"""
    READ = 0      # 读取操作 — 自动允许，无需审批
    WRITE = 1     # 写入操作 — 需要用户确认
    EXECUTE = 2   # 执行操作 — 需要用户严格确认（醒目提示）
```

权限映射：

| 工具 | 权限等级 | 说明 |
|:-----|:---------|:-----|
| `read_file` | READ | 读取文件，自动允许 |
| `glob` / `grep` | READ | 搜索操作，自动允许 |
| `write_to_file` | WRITE | 写入文件，需审批 |
| `str_replace_editor` | WRITE | 编辑文件，需审批 |
| `bash` | EXECUTE | Shell 命令，需严格审批 |
| `web_fetch` | WRITE | 网络请求，需审批 |

### 2.4 等待机制的实现

- [ ] 使用 `asyncio.Event` + `asyncio.wait_for()` 实现审批等待：

```python
class PendingRequest:
    request: PermissionRequest
    event: asyncio.Event      # 用于等待用户响应
    result: PermissionResult | None = None
    
    async def wait(self, timeout: Optional[float] = None) -> PermissionResult:
        """等待用户审批，可设置超时"""
        try:
            await asyncio.wait_for(self.event.wait(), timeout=timeout)
        except asyncio.TimeoutError:
            return PermissionResult.TIMEOUT
        return self.result
```

- [ ] 超时：默认 300s（5 分钟），超时视为拒绝
- [ ] 可中断：用户发送 `cancel` 信号时也视为拒绝

## 5. 注意事项

- 等待模型使用 `asyncio.Event` —— 这是 Python 异步等待的标准方式
- **永远不要**绕过审批直接执行 WRITE/EXECUTE 操作
- 审批超时要合理——太短让用户着急，太长让 AI 空等
- "始终允许"默认不持久化到磁盘，用户可选（Phase 3 实现）
- 所有审批事件必须可追溯，便于调试和安全审计
