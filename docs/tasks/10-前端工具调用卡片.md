# 任务 10：前端工具调用卡片与审批 UI

> **所属阶段**：Phase 2 - 真实工具落地与权限系统 (The Action Layer)
> **对应模块**：模块 1 (UI) — Tool Cards / Approval Dialogs
> **预估工时**：2-3 天
> **依赖**：任务 02 (前端骨架)、任务 03 (WebSocket)、任务 07 (FileSystem)、任务 08 (Shell)、任务 09 (Permission)

---

## 1. 任务目标

开发前端工具调用可视化组件：当 AI 调用工具时，在消息流中以"卡片"形式展示操作详情（文件 diff 预览、Shell 命令展示），并嵌入审批按钮（Approve / Reject / Always Allow）。

### 2.6 状态管理集成

- [ ] 在 React 状态（`useState` / `useReducer`）中维护工具调用状态
- [ ] `pending_tool_calls: Map<string, ToolCallState>`
- [ ] WebSocket 消息分发更新：

```typescript
const handleToolCallStart = (msg) => {
  // 添加新卡片
  setPendingCalls(prev => new Map(prev).set(msg.call_id, {
    toolCall: msg,
    status: msg.level === 'read' ? 'approved' : 'pending',
  }));
};

const handlePermissionRequired = (msg) => {
  // 更新卡片为等待审批状态
  setPendingCalls(prev => {
    const updated = new Map(prev);
    updated.set(msg.request_id, { ...updated.get(msg.request_id), status: 'pending' });
    return updated;
  });
};

const handleToolCallResult = (msg) => {
  // 更新卡片为完成状态
};
```

## 5. 注意事项

- 权限审批 UI 是安全流程的一部分——拒绝按钮不能太小或难找
- EXECUTE 级别的二次确认是必备的，防止误触危险操作
- 卡片状态必须可靠——不要出现"批准了却还在等待"的 bug
- 保持卡片设计紧凑，不要占用过多对话空间
- 移动端需要适配：卡片在小屏幕上应完整可操作
