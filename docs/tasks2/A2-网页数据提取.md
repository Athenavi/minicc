# A2：网页数据提取工具

> **所属阶段**：Phase A - Web 自动化
> **预估工时**：3 天
> **依赖**：A1（浏览器控制）

---

## 1. 任务目标

实现网页数据的结构化提取能力，让 AI 能从页面中提取表格、列表、JSON 数据、特定文本内容等。对标影刀的"网页数据抓取"能力。

## 2. 详细子任务

### 2.1 数据提取工具集

- [ ] `tools/web/scraper.py`

#### `WebExtractTableTool` — 提取表格数据

```python
class ExtractTableInput(BaseModel):
    selector: str = Field(description="表格的 CSS 选择器")
    page_id: str | None = None
    include_headers: bool = True
    max_rows: int = Field(default=100, ge=1, le=10000)
```

输出结构化数据（Markdown 表格 + JSON 两种格式）。

#### `WebExtractListTool` — 提取列表数据

```python
class ExtractListInput(BaseModel):
    selector: str = Field(description="列表项容器的 CSS 选择器")
    item_selector: str = Field(description="列表项的 CSS 选择器")
    page_id: str | None = None
    max_items: int = Field(default=50, ge=1, le=1000)
```

#### `WebExtractTextTool` — 提取指定文本

```python
class ExtractTextInput(BaseModel):
    selector: str = Field(description="目标元素选择器")
    page_id: str | None = None
    attribute: str | None = Field(default=None, description="提取属性而非文本")
```

#### `WebExtractJsonTool` — 提取 JSON-LD / 结构化数据

- 自动发现页面中的 `script[type="application/ld+json"]`
- 提取 JSON 格式的页面元数据

### 2.2 AI 驱动的智能提取

- AI 分析页面结构后自动推荐提取策略
- 用户说"提取这个表格" → AI 自动定位最近的 table 元素
- 支持"先截图再看"模式：AI 先截图页面区域，再提取数据
