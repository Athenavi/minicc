# A1：Playwright 浏览器控制工具

> **所属阶段**：Phase A - Web 自动化
> **预估工时**：5 天
> **依赖**：V0.1 基础工具链（ToolRegistry, PermissionSystem, SSE）

---

## 1. 任务目标

实现基于 Playwright 的浏览器控制工具组，让 AI 能通过自然语言指令完成：打开浏览器、导航到 URL、点击元素、输入文本、截图、获取页面信息等核心操作。

对标影刀"浏览器控制"能力，但 AI Native 优势：AI 可理解页面语义，自动选择合适的选择器。

## 2. 详细子任务

### 2.1 Playwright 浏览器管理器

- [ ] `tools/web/browser.py` — `BrowserManager` 类
  - `async start(headless=True)` — 启动浏览器实例
  - `async new_page(url=None)` — 创建新标签页
  - `async close()` — 关闭浏览器
  - `async screenshot()` — 截取当前页面截图
  - `async get_cookies()` / `async set_cookies()` — Cookie 管理
  - 多实例支持（多个独立浏览器 Context）
  - 浏览器实例池（复用已打开的浏览器）

```python

        """导航到 URL"""
```

### 2.2 浏览器控制工具集

#### `BrowserNavigateTool` — 导航

```python
class BrowserNavigateInput(BaseModel):
    url: str = Field(description="要打开的 URL")
    page_id: str | None = Field(default=None, description="标签页 ID（默认当前页）")
    wait_until: str = Field(default="load", description="等待条件: load/domcontentloaded/networkidle")
```

#### `BrowserClickTool` — 点击元素

```python
class BrowserClickInput(BaseModel):
    selector: str = Field(description="CSS 选择器或 XPath")
    page_id: str | None = None
    wait_before: float = Field(default=0.5, ge=0, le=5)
```

#### `BrowserFillTool` — 输入文本

```python
class BrowserFillInput(BaseModel):
```

- `BrowserNavigateTool` → `PermissionLevel.WRITE`
- `BrowserClickTool` → `PermissionLevel.EXECUTE`
- `BrowserFillTool` → `PermissionLevel.EXECUTE`
- `BrowserGetHtmlTool` → `PermissionLevel.READ`
- `BrowserScreenshotTool` → `PermissionLevel.READ`

## 5. 注意事项

- Playwright 首次使用需 `playwright install` 安装浏览器
- headless 模式默认开启，可通过配置关闭（调试时）
- 每个 session 独享一个 BrowserContext，互不干扰
- 页面截图传给 Vision 模型时注意 token 消耗
- 选择器策略优先级：data-testid > id > CSS > XPath > text
